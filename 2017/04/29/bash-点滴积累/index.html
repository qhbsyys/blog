引号区别

在字符串中引用一个变量的值时，要使用双引号——单引号中不允许解释变量引用
使用双引号除了变量名[2]前缀($)、后引符(`)和转义符(\)外，会使shell不再解释引号中其它所有的特殊字符
在echo语句中，只有句子分割和保存空白符的时候，才需要把参数用双引号引起来

单引号，在单引号内，除了单引号本身，每个特殊字符都只是字面的意思。在一双单引号里再加单引号是不行的。

sed

sed 主要是以行为单位进行处理，将数据行进行替换、删除、新增、选取等操作。命令格式：
sed [-nefri] 'command' 文本或文件名

d : 删除一行

sed ‘3d’ file  #删除第3行
sed ‘3,$d’ file  #删除第3到最后一行

s : 替换字符串 支持正则

sed ‘s/abc/def/g’ file  #把行内的所有abc替换成def，如果没有g，则只替换第一个
sed ‘s/abc/&def/g’ file  #在所有的abc后面添加def（&表示匹配的内容）
sed ‘5s/abc/def/g’ file  #只对第5行进行替换操作

a : 在指定位置后插入一行 （i :在指定位置之前插入一行）

sed ‘/abc/a\def’ file  #在包含abc的行后插入一行，写入def

c : 替换指定行

sed ‘/abc/c\def’ file  #将包含abc的行替换成def
#c参数在mac下总是报错，可以使用s参数达到相同的效果 sed ‘s/.abc./def/‘

n : next的意思 sed命令中可以使用{}将多条命令封装起来，用;间隔

sed ‘/abc/{n; s/aaa/bbb/g;}’ file  #读取包含abc的行的下一行，替换所有的aaa为bbb。

q : quit

sed ‘/abc/{s/aaa/bbb/;q;}’ file  #找到包含abc的一行，把aaa替换成bbb，然后退出sed

grep

grep [-acinv] [--color=auto] '搜寻字符串' filename

-a ：将 binary 文件以 text 文件的方式搜寻数据
-c ：计算找到 ‘搜寻字符串’ 的次数
-i ：忽略大小写的不同，所以大小写视为相同
-n ：顺便输出行号
-v ：反向选择，亦即显示出没有 ‘搜寻字符串’ 内容的那一行
--color=auto ：将找到的关键词部分加上颜色显示
